비개발자·시각 학습자 특성 + CLI 16개 외우기 어려움 + 분류·시각화 필요 → 기존 11~15번 중 UI(15번)를 1순위로 올림. 로컬 웹 대시보드 + PWA + 작업 스케줄러 자동 시작 + 반응형(모바일 대응) 방식.

## 배경
- 사용자 백요한은 비개발자·비전공자. CLI 16+개 중 2~3개만 기억.
- 시각 학습자: 도식·마인드맵·조직도로 이해가 가장 빠름.
- '도서관처럼 분류별로 꺼내쓰는 구조' 선호.

## 결정
- 원래 5순위였던 UI를 1순위로 상향.
- 형태: 로컬 웹 대시보드 (localhost) + PWA (바탕화면 앱처럼).
- 작업 스케줄러에 서버 자동 시작 등록.
- 반응형 디자인으로 모바일(같은 와이파이) 대응.
- 집 컴퓨터/노트북 양쪽에서 git pull 후 동일 대시보드 사용.

## 대안 고려
- Tauri(.exe): 개발 비용 큼 → 나중에 필요시 전환 가능.
- 클라우드 배포: 현재 불필요 → 외부 접속 필요시 Cloudflare Tunnel 등 추가.
