Node 프로세스 폭증/무한 로딩 재발 방지를 위해 실행 표준(4000 고정), 3분 복구 절차, 디버깅 체크리스트를 SoT 규칙으로 고정

## 배경

- 로컬 대시보드 실행 중 Node 프로세스 급증, 무한 로딩, `tailwindcss resolve` 오류, `EADDRINUSE`가 반복됨.
- 임시 대응만 반복하면 같은 장애가 재발할 가능성이 높음.

## 결정

1. `memory/rules/dashboard-runtime-stability.md`를 신규 생성해 운영 표준으로 사용.
2. 실행 스크립트 표준을 루트 `package.json`에 추가 (`dashboard:dev:4000`, `dashboard:start:4000`).
3. 장애 시 복구 절차를 `taskkill -> .next 정리 -> build/start`로 고정.
4. 하네스 참조표(`memory/rules/agent-harness.md`)와 스펙(`docs/DASHBOARD-SPEC.md`)에 해당 규칙 링크를 연결.

## 기대 효과

- 1인 운영에서 재현 가능한 복구 루틴 확보
- dev 워처 중복 실행으로 인한 Node 폭증 빈도 감소
- 동일 장애 발생 시 원인/조치 기록 일관성 확보
