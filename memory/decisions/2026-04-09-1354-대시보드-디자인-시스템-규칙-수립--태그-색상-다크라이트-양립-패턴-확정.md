태그 뱃지 색상을 dark:/기본 분리 구조(PALETTE 패턴)로 확정. 디자인 규칙 문서(memory/rules/dashboard-design-system.md)를 SoT로 생성.

## 배경

- v1 태그 색상이 `text-*-400` 단일 값이라 라이트 모드에서 거의 안 보이는 문제 발생.
- 다크 모드에서도 300 톤으로 대비 부족 피드백.
- 디자인 결정·패턴을 기록하는 규칙이 없어서 같은 실수 반복 위험.

## 결정

1. **PALETTE 패턴 도입**: `doc-card.tsx`에 `TagStyle` 타입 + 19개 색상 팔레트. 라이트(`text-*-700 bg-*-100`) / 다크(`text-*-300 bg-*-500/25`) 분리.
2. **디자인 시스템 규칙**: `memory/rules/dashboard-design-system.md` — 색상, 타이포, 간격, 반응형, 컴포넌트 패턴, 디자인 변경 체크리스트.
3. **디자인 변경 시 기록 원칙**: 색상·레이아웃·패턴 변경은 `append_decision`으로 기록.

## 대안 기각

- CSS 변수로 태그 색 정의 → Tailwind utility 체계와 맞지 않고, 관리 오버헤드.
- 단일 값 + opacity 조정만 → 한쪽 모드에서 반드시 깨짐.

## 영향

- 이후 대시보드 디자인 작업은 `dashboard-design-system.md` 참조 필수.
- 새 태그 색상 추가 시 PALETTE에 먼저 등록.
