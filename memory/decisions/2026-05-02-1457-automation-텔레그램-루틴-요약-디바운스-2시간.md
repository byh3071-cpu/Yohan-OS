automation-batch의 routine 완료 요약 텔레그램 알림은 기본 2시간에 1회로 제한하고, `review/failed`가 있을 때만 즉시 알림한다.

## 배경

- 대시보드 자동화 배치는 30분 간격으로 도는데, 매 사이클 텔레그램 완료 알림이 들어와 1인 운영자가 알림 자체에 둔감해짐.
- 정작 사람이 봐야 할 `review > 0` / `failed > 0` 신호가 routine 노이즈에 묻힘.

## 결정

1. `scripts/automation-batch.ts`에 `ROUTINE_NOTIFY_INTERVAL_MS = 2h` 도입.
2. 마지막 routine 알림 시각은 `memory/metrics/automation/last-telegram-notify.json`에 저장(파일 부재 시 즉시 발송 가능).
3. `review > 0 || failed > 0` 인 사이클은 인터벌 무시하고 **항상** 텔레그램 발송.
4. `--dry-run` 모드에서는 마지막 시각 파일을 갱신하지 않는다 (실 발송 여부와 별개).

## 근거

- 1인 운영자가 무의미한 routine 요약 알림에 둔감해지는 비용 > routine 알림 누락 위험.
- `review/failed`는 즉시 채널 유지로 균형. 인터벌은 환경 변수화하지 않고 코드 상수로 두는 게 운영 단순.

## 기대 효과

- 30분마다 오던 의미 없는 "스캔=N 처리=0" 알림이 2시간에 1회로 축소.
- 검토·실패 알림은 그대로 즉시 도달 → 신호 대비 잡음 비율 개선.
