# 노션 동기 — 규칙 (양방향 + SoT 우선)

`memory/`(자체 SoT)와 노션을 **둘 다** 쓰되, **항상 `memory/`가 진실의 원천(SoT)** 이다.

---

## 1. 최상위 원칙 (불변)

1. **충돌 시 SoT 승리**
  같은 사실·결정·요약이 `memory/`와 노션에서 다르면 **무조건 `memory/`를 따른다.** 노션은 **나중에 SoT에 맞춰 고친다** (또는 SoT를 먼저 고친 뒤 노션만 반영).
2. **에이전트 런타임**은 MCP·`get_context`로 읽는 `**memory/`만 신뢰**한다. 노션은 **사람·협업·뷰**용이다.
3. **비밀**은 노션·SoT 모두 **평문 금지** (기존 `profile.yaml` must_not과 동일).

---

## 2. 쓰기 (memory → 노션)

1. 노션으로 가는 것은 **요약 카드 수준** (제목, 한 줄 요약, 링크·파일 경로, 태그)을 기본으로 한다.
2. 푸시 전에 **SoT에 이미 반영된 내용**만 노션에 올린다 (인제스트·결정이 `memory/`에 먼저 있음).
3. 노션 페이지/DB ID·필드 매핑은 **환경변수 또는 설정 한 곳**에만 둔다 (키는 커밋하지 않음).

---

## 3. 읽기 (노션 → memory)

1. 노션에서 가져온 것은 **“제안/큐”** 로 취급한다. **SoT에 병합할 때는** `append_decision`·파일 수정·또는 명시적 “승인” 절차를 거친다.
2. **자동 덮어쓰기 금지**: 노션만 바뀌었다고 `profile.yaml`·`decisions/`를 **자동으로 덮어쓰지 않는다.**
3. 주기적 읽기(동기)는 **최소 간격**을 둔다 (예: 수동 트리거 또는 1일 1회) — 레이트 리밋·충돌 방지.

---

## 4. 추가로 두면 좋은 규칙


| 규칙         | 이유                                                                                          |
| ---------- | ------------------------------------------------------------------------------------------- |
| **타임스탬프**  | SoT 쪽 `ingested_at` / `updated_at`과 노션 `last_edited`를 비교해 **어느 쪽이 최신인지** 판단 (충돌 해소 보조).     |
| **범위 고정**  | “노션에서 읽을 DB/페이지 ID” **화이트리스트**만 둔다 — 전 워크스페이스 읽기 금지.                                        |
| **한 방향 큐** | 읽기 결과는 먼저 `memory/inbox/notion-queue.md` 같은 **큐 파일**에만 쌓고, 사용자 또는 Evaluator 통과 후 본 SoT에 반영. |
| **삭제 정책**  | 노션에서 행 삭제 시 SoT는 **자동 삭제하지 않음** (보관·아카이브만). SoT 삭제는 **명시적 명령**만.                            |
| **노션 AI**  | 노션 내 AI가 만든 문장은 **SoT 반영 전** P/G/E 또는 최소한 **사람 확인** 한 줄.                                    |


---

## 5. 구현 시 체크리스트 (나중에 붙일 MCP/스크립트용)

- 환경 변수 이름·`.env.example` — 레포 루트 참고 (실제 값은 `.env`만, 커밋 금지)
- 풀 큐 고정 경로: `memory/inbox/notion-queue.md` (`NOTION_QUEUE_FILE`로 오버라이드 가능). `get_context` → `notion_queue`에 미리보기
- `push`: 요약만, idempotent 키(예: SoT 파일 경로 해시)로 **같은 카드 갱신**
- `pull`: 큐 파일에만 append, **merge는 명시**

---

**텔레그램 OCR → 리소스/서머리 DB** 별도 절차·삭제 정책: `memory/rules/notion-ocr-pipeline.md` (SoT는 여전히 `memory/ingest/insights/` 우선).
