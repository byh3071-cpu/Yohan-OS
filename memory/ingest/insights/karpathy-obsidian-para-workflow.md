# Karpathy 옵시디언 흐름에서 병목 → PARA로 수렴

## 핵심

- **문제 의식:** 지식·캡처가 쌓이면 정리·재검색·업데이트·백링크·개발 시 재사용이 부담이 됨.
- **구체 병목:** (1) 정리 과다 (2) 같은 것 재구글 (3) 반복 질문 (4) 전역 업데이트 귀찮음 (5) 백링크 미활용 (6) 개발 맥락으로 끌어오기 어려움.
- **결론:** 기존 사례 조사 + 수일 브레인스토밍 끝 **PARA Method** 채택.

## 캡처 페어 (인박스/노션)

- **실행 순서(2/2 카드):** raw → Inbox, 레거시 노트 → 헌법에 맞게 리팩토링, Karpathy식 구조로 더 정교하게 — `message_id` **20**, **34**. 노션 일괄 푸시에 포함됨.

## 적용

- 볼트에 PARA 변형 **대분류**를 고정하고 인박스만 저마찰로 받기 — 세부는 `obsidian-vault-para-system-guide.md`.
