# AI가 UX를 설계하는 방식: 화면보다 흐름이 먼저인 ‘Figr’ 

최근 몇 년 사이 AI 디자인 도구는 폭발적으로 늘어났습니다. 텍스트 프롬프트만 입력하면 UI 화면을 만들어주거나, 와이어프레임을 자동으로 생성해 주는 서비스를 이제는 쉽게 만나볼 수 있죠. 하지만 실제 제품을 만드는 기획자나 디자이너 입장에서 이런 도구를 쓰다 보면, 한 가지 공통적인 아쉬움을 느끼게 됩니다. 바로 "화면은 빠르게 만들어지는데, UX 설계와 제품 사고는 여전히 사람이 해야 한다"는 점입니다. 이 지점에서 등장한 서비스가 바로 오늘 소개할 ‘Figr’입니다. Figr는 스스로를 단순한 AI 디자인 툴이 아니라 "AI Design Agent"라고 설명합니다. 화면을 만들어주는 도구라기보다는, UX 사고 → 사용자 흐름 설계 → 프로토타입 제작까지 이어지는 제품 설계 과정을 AI가 함께 수행하는 협업 도구에 가깝습니다.
