# DB-120 별자리 D-4 성능 스파이크 메모

## 측정 관점 (코드 정적 분석)

| 요소 | 구현 | 비용 메모 |
|------|------|-----------|
| 배경 별 | `<Stars count={3200~4200}>` | GPU 포인트 스프라이트, 저배포에서 주된 fillrate 부담 가능 |
| 엣지 | `LineSegments` + `useFrame` 버퍼 갱신 | 엣지 수 E에 비례해 매 프레임 CPU write |
| 노드 | 메시별 별 + Html 툴팁 | Html은 드로우 수 적지만 DOM 레이아웃 비용 |
| 성운 D-4 | 카테고리당 저폴리 구 + Additive | 메시 6~9개, 저비용 |
| Fog | `fogExp2` | 픽셀당 약간의 비용 |

## 단계적 도입 제안

1. **유지**: 옵션 C 레이아웃·연결선·토글 가능한 성운/포그.
2. **부하 시**: `Stars` `count`를 1600 근처로 낮추거나 `nebula` OFF 시 4200 유지(현재 분기 있음).
3. **확장(~1000 노드)**: 엣지 LOD(줌아웃 시 얇은 세트만)·Html 라벨 거리 컷 검토.

## 결정 (이번 스파이크)

- **인스턴스드 메시 전환**: 현재 노드 수에서는 필수 아님. 노드 300+ 또는 저사양 기기에서 재측정 후 검토.
- **포그 밀도**: `0.009` 유지; 성능 이슈 시 `nebula` 토글로 사용자가 우회 가능.

실측 FPS는 브라우저 Performance 패널에서 별자리 탭 전환 후 기록하는 것을 권장한다.
